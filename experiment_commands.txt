## All practical ways to run it

1. Dry-run only (no algorithms executed, just enumerate runs)
```bash
python experiment_sweep.py --dry-run
```

2. Smoke test (tiny, fast sanity run)
```bash
python experiment_sweep.py --profile smoke
```

3. Full paper sweep (default profile is `large`)
```bash
python experiment_sweep.py --profile large
```

4. Resume interrupted sweep
```bash
python experiment_sweep.py --profile large --resume
```

5. Limit number of runs (great for debugging)
```bash
python experiment_sweep.py --profile large --max-runs 20
```

6. Skip sensitivity grid in large profile
```bash
python experiment_sweep.py --profile large --no-sensitivity
```

7. Custom output folder
```bash
python experiment_sweep.py --out-dir outputs/my_paper_sweep
```

8. Run only selected implementations
```bash
python experiment_sweep.py --methods masked_ksvd,multiscale_ksvd,crown_full
```

Allowed methods:
- `biharmonic`
- `zero_ksvd`
- `masked_ksvd`
- `multiscale_ksvd`
- `crown_full`
- `crown_no_nonlocal`
- `crown_sparse_only`
- or `all`

9. Restrict images/masks
```bash
python experiment_sweep.py --images brick,camera --masks random_p30,square_s24
```

10. Hyperparameter sweeps (the key part for your paper)
```bash
python experiment_sweep.py \
  --methods masked_ksvd,multiscale_ksvd,crown_full,crown_no_nonlocal,crown_sparse_only \
  --images brick,camera,coins,moon \
  --masks random_p10,random_p30,random_p50,square_s24,scratches_n4_t8 \
  --seeds 42,69,123 \
  --K 128,256,384 \
  --sparsity 4,8,12 \
  --dict-iters 10,25,40 \
  --n-train 1000,3000 \
  --crown-T 1,3,5,8 \
  --out-dir outputs/paper_main_sweep
```

11. Verbose internals (prints inner K-SVD/CROWN logs)
```bash
python experiment_sweep.py --profile large --verbose
```

## Where outputs go

By default into:
- `outputs/experiment_sweeps/runs.csv`
- `outputs/experiment_sweeps/aggregate.csv`
- `outputs/experiment_sweeps/manifest.json`
- plots in `outputs/experiment_sweeps/plots/`

## Suggested run strategy (reliable workflow)

1. `--dry-run` first to check total combinations.
2. `--profile smoke` to verify pipeline end-to-end.
3. Main custom sweep with your full grid.
4. If interrupted, rerun with `--resume`.
